Описание
Реализация алгоритма поиска с возвратом (backtracking) для решения судоку размером N×N, где N может принимать значения от 3 до 9. Программа считывает частично заполненное судоку из файла и пытается найти его решение методом систематического перебора с оптимизацией выбора следующей клетки.

Реализованные операции
Создание и инициализация структуры судоку

Проверка возможности размещения числа в клетке

Размещение числа в клетке с обновлением состояния

Отмена размещения числа (возврат)

Выбор оптимальной клетки для следующего хода (минимальное количество вариантов)

Рекурсивный поиск решения

Валидация входных данных

Вывод решения в файл

Сборка и запуск
Проект создан в Visual Studio 2022. Для сборки и запуска:

Открыть проект в Visual Studio

Нажать Ctrl+Shift+B для сборки

Поместить входной файл input.txt в директорию проекта

Нажать Ctrl+F5 для запуска программы

Результат будет записан в output.txt

Формат входного файла:

text
N
значения матрицы N*N×N*N (0 для пустых клеток)
Тестирование
Реализовано 11 юнит-тестов, покрывающих основные функции и граничные случаи:

test_valid_creation_no1 - проверка корректного создания структуры

test_invalid_sizes_no2 - проверка обработки недопустимых размеров

test_invalid_coordinates_no3 - проверка обработки некорректных координат

test_invalid_value_no4 - проверка обработки недопустимых значений

test_row_conflict_no5 - проверка обнаружения конфликтов в строке

test_zero_clears_no6 - проверка очистки клетки нулем

test_known_puzzle_no7 - решение классического судоку 9×9

test_unsolvable_no8 - проверка неразрешимого судоку

test_display_count_no9 - проверка корректности вывода

test_edge_cases_no10 - проверка граничных случаев (NULL, некорректные значения)

test_sudoku_4x4_no11 - решение судоку 4×4

Все тесты проходят успешно.

Особенности реализации
Структуры данных
Puzzle - основная структура, содержащая:

size: размер блока (N)

dimension: общий размер (N*N)

cells: массив значений клеток

rows/cols/boxes: битовые массивы для быстрой проверки конфликтов

consistent: флаг согласованности данных

Алгоритмические решения
Оптимизация выбора клетки - выбор клетки с минимальным количеством возможных вариантов (эвристика MRV)

Битовые массивы - использование массивов unsigned char для O(1) проверки конфликтов

Рекурсивный возврат - классический backtracking с восстановлением состояния

Преимущества и недостатки алгоритма поиска с возвратом
Преимущества:

Гарантированно находит решение, если оно существует

Простота реализации и понимания

Не требует дополнительной памяти для хранения промежуточных состояний (кроме стека вызовов)

Легко модифицируется для других комбинаторных задач

Недостатки:

Экспоненциальная сложность в худшем случае

На больших размерностях (N > 9) становится непрактичным

Эффективность сильно зависит от эвристик выбора следующего хода

Глубина рекурсии может быть большой (до 81 уровня для судоку 9×9)
