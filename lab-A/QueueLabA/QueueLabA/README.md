Лабораторная работа А — Вариант 11
Очередь (реализация на связном списке и массиве)

Цель работы:
Реализовать две версии очереди: на основе связного списка и на основе массива. Провести сравнение по принципу работы, скорости и удобству использования.

Средства разработки:
Язык программирования: C
Среда: Visual Studio
Тестирование: Google Test

Описание работы:
Обе реализации поддерживают одинаковые функции:
enqueue, dequeue, peek, is_empty, is_full (для массива) и destroy.
некоторые функции возвращают параметр status:
1 — операция выполнена успешно,
0 — ошибка (очередь пуста или переполнена).

Сравнение:
Очередь на связном списке гибкая и не имеет ограничения по размеру, но медленнее из-за динамического выделения памяти.
Очередь на массиве быстрее, но имеет фиксированный размер и требует реализации циклического обращения к элементам.

Тестирование:
Проверялись:
создание и пустое состояние;
добавление и удаление элементов;
порядок FIFO;
работа функций peek, is_empty, is_full;
поведение при пустой и полной очереди.
Все тесты успешно пройдены

Вывод:
Обе очереди работают по принципу FIFO (первым пришёл — первым вышел).
Реализация на списке удобна при неизвестном размере данных,
а реализация на массиве эффективнее при фиксированном объёме и ограниченном количестве элементов.
