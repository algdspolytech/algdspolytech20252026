## Вариант 11: Очередь
* Реализуйте очередь с помощью связного списка
* Реализуйте очередь с помощью массива
* Сравните полученные реализации, в чем преимущества и недостатки каждой?

---

## Очередь на основе массива
### Преимущества
* Локальность данных: элементы хранятся в непрерывной области памяти
* Производительность за счет отсутствия накладных расходов на выделение и освобождение памяти при операциях
* Простая реализация, которая обеспечивается минимальным количеством указателей для управления, не составляет труда обращение к элементам по индексу
### Недостатки
* Ограниченный размер очереди, фиксируемый при инициализации
* Сложность управления памятью
* Невозможность расширяемости

## Очередь на основе связного списка
### Преимущества
* Доступна расширяемость
* Нет ограничений на максимальный размер
* Нет сложности в управлении с памятью
### Недостатки
* Отсутствует локальность данных, нет структуризации элементов в памяти
* Расходы на дополнительные указатели
* Сложность реализации в связи с риском ошибок при работе с указателями и необходимостью управления связями между узлами.
